\section[Egress Stage]{Egress Stage}

Un Egress stage rappresenta il punto terminale della pipeline di \textbf{Goccia}, responsabile dell'esportazione dei messaggi elaborati verso destinazioni esterne al sistema. Diversamente dai Processor stage, che trasformano messaggi e li inoltrano a stage successivi, gli Egress stage ricevono messaggi da uno stage precedente tramite un connector di input e li materializzano verso sink esterni quali database, broker di messaggistica, file system o socket di rete. In altri termini, ogni Egress stage implementa una funzione di output: \(f(\text{messaggio\_in}) \rightarrow \text{void}\), terminando cos√¨ il flusso di elaborazione e rendendo i dati disponibili per sistemi downstream o per la persistenza.
